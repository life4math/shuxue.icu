# shuxue.icu 开发计划

更新日期：2026-07-24

## 产品主线

平台后续围绕一条可持续积累的数据链路开发：

1. 教师维护知识图谱和知识正文；
2. 教师上传教辅、试卷和个人经验，AI 形成待审核内容；
3. 审核后的知识、题目、方法进入正式数据库；
4. 备课工作台引用这些正式内容，形成可发布、可演示的讲义；
5. 使用数据和教师修订继续丰富知识、题目、方法与教学经验。

公开演示数据与教师正式数据继续隔离。教师后台使用账号、权限、审计和发布快照；公开站点只读取已经发布的不可变快照。

## 已完成

### 阶段 0：平台与部署基线

- 正式数据库、账号与角色权限；
- 登录限流、会话与 CSRF 防护；
- AI 上传任务、失败重试和审核；
- 健康检查、数据库迁移、部署门禁、备份与失败回滚；
- Python 3.11 生产运行时和可复现依赖。

### 阶段 1：动态知识图谱

- 知识节点永久 ID、稳定代码和历史别名；
- 层级、关系、版本、归档与合并；
- 知识正文草稿、审核和发布快照；
- 公开网站从正式知识 API 读取，静态内容只作为不可用时的演示回退。

### 阶段 2A：在线备课一期

- 课程与课次管理；
- 正文、KaTeX 公式、提示、例题、题目引用、知识点引用和图片内容块；
- 大纲、表单编辑和实时预览三栏工作台；
- 3 秒防抖自动保存、30 秒兜底保存、浏览器临时备份；
- 乐观锁冲突保护和不可变版本记录；
- 草稿、待审核、发布、撤下和回滚；
- 公开讲义目录、阅读页和课堂演示模式；
- 后台预览与公开页面复用同一个内容块渲染器。

## 后续开发顺序

### 阶段 2B：正式题库与引用闭环

目标：让讲义中的题目引用不再依赖手工输入。

- 建立题目、题目版本、来源、知识点关联和发布快照；
- 把现有 AI 审核结果正式写入题库；
- 在备课工作台提供题目搜索、筛选、预览和插入；
- 插入讲义时保存题目 ID，并在发布快照中冻结必要展示字段；
- 补充题目去重、合并、归档和审计。

验收：新增题目后可被备课搜索；题目后续修订不会悄悄改变历史已发布讲义。

### 阶段 3：教辅经验库与动态积累

目标：把不同教辅书对知识点和题目的总结保存为可审核、可追溯的经验。

- 建立来源、文档片段、经验条目、证据引用和置信度；
- 经验条目关联知识点、题目、方法和适用年级；
- 同一结论支持多个来源，保留相同、补充和冲突关系；
- 教师可以修订、合并、采纳或驳回 AI 提取结果；
- 知识详情显示“正式结论”和“来源经验”，不混成一段不可追溯文本。

验收：每条经验都能追溯到来源页码或片段，AI 生成内容必须经教师审核才能进入正式数据。

### 阶段 4：备课智能编排

目标：利用持续增长的知识、题目和经验，协助教师生成可编辑的讲义草稿。

- 根据课时、学生水平和教学目标生成大纲建议；
- 按知识先修关系、难度和教学节奏推荐题目；
- 从经验库推荐易错点、典型方法和多来源比较；
- AI 只创建草稿和修改建议，不直接发布；
- 记录教师采纳、修改和拒绝，用于改进检索与排序。

验收：系统给出建议时同时返回引用的知识、题目和经验 ID，教师可以逐项接受。

### 阶段 5：课堂与学习反馈

- 课堂演示中的分步揭示、计时和临时批注；
- 讲义阅读进度与匿名聚合反馈；
- 将易错反馈关联到知识点和题目；
- 为教师提供讲义版本效果比较和改版建议。

## 本期明确不做

- 不直接把 PPT、Word 转换结果写入已发布讲义；
- 不做多人同时编辑和复杂协同锁；
- 不把公开原型中的虚构学生数据迁入正式数据库；
- 不引入新的前端框架或构建系统；
- 不让静态 JSON 与数据库同时成为可写的事实来源。
