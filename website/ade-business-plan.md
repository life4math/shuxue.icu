# shuxue.icu — ADE 组合收费方案设计

## 1. 模型选择逻辑

**A+D+E 组合的核心协同：**

| 模型 | 角色 | 目标客户 | 收入类型 |
|------|------|----------|----------|
| D · Freemium | 获客引擎 | 个体教师 | 0 → 间接转化 |
| A · SaaS 订阅 | 核心收入 | 教师/教研组 | 月付订阅 |
| E · 数据 API | 利润池 | EdTech/研究者 | 按量计费 + 项目制 |

**三层闭环：**
- D 免费层吸引教师 → 使用行为产生数据 → 数据沉淀为资产
- 教师升级 A Pro/Team → 稳定月付现金流 → 覆盖运营成本
- E API 将脱敏聚合数据变现 → 高客单价 → 贡献 60%+ 利润率
- E 的收入反哺 D 的免费成本 → 可持续免费获客

## 2. 定价层级详细设计

### 2.1 FREE 层（D · Freemium）

**定位**：零门槛获客，降低教师试用成本

| 功能 | 限制 | 设计意图 |
|------|------|----------|
| 题库浏览 | 10 题/日 | 感受题库质量，但不足以备课 |
| 知识图谱 | 仅查看 | 展示产品深度，制造升级欲望 |
| 学情分析 | 3 次/月 | 体验 AI 分析能力，刚需频次不足 |
| AI 审核 | 无 | 强差异化功能，Pro 核心卖点 |
| 方法模板 | 仅查看 | 感受方法体系，无法自定义 |
| 文件上传 | 1 次/日 | 轻度体验，批量操作需升级 |
| Blog | 完整访问 | 内容营销工具，不限制（引流） |
| 数据导出 | 无 | Pro 独占功能 |

**免费层成本估算**（每人/月）：
- 存储 + CDN：≈ ¥0.5
- AI 分析 3 次（LLM API）：≈ ¥1.5
- 基础服务：≈ ¥0.5
- **总计 ≈ ¥2.5/用户/月**

500 用户 = ¥1,250/月 → 由 Pro 和 API 收入覆盖

### 2.2 PRO 层（A · SaaS · ¥49/月）

**定位**：个人教师全功能解锁，AI 驱动教研

**定价锚点分析**：
- 学而思教师版 ¥39/月（仅题库浏览）
- 好未来教研云 ¥68/月（题库 + 基础分析）
- shuxue.icu ¥49/月：题库 + AI 审核 + 知识图谱 + 方法体系 — **三合一性价比**

| 功能 | 规格 | 核心卖点 |
|------|------|----------|
| 题库 | 无限浏览 + 创建 + 编辑 | 备课刚需 |
| 知识图谱 | 查看 + 节点注解 | 独家差异化 |
| 学情分析 | 无限次 + PDF 导出 | 替代手工统计 |
| AI 审核 | 无限 · 优先队列 | **最强卖点** |
| 方法模板 | 查看 + 创建自定义 | 教学结构化 |
| 文件上传 | 批量 10 次/日 | 效率提升 |
| 数据导出 | CSV + PDF | 报告交付 |
| Blog | 完整 + 草稿编辑 | 个人品牌建设 |

**Pro 层利润率**：
- 收入：¥49/月
- 成本：AI 审核无限 ≈ ¥8/月 + 存储 ≈ ¥1 + 服务 ≈ ¥2
- **毛利 ≈ ¥38/月 · 毛利率 ≈ 78%**

### 2.4 API 层（E · 数据服务 · ¥500+/月）

**定位**：结构化教研数据变现，高客单价利润池

| 产品 | 价格 | 说明 |
|------|------|------|
| 标准查询 API | ¥500/月最低 + ¥0.01/次调用 | 题库/知识图谱/解题路径 |

**API 利润率**：
- 标准 API：毛利 ≈ 85%（边际成本趋零）

- 定制服务：毛利 ≈ 50%（人力密集）

## 5. 技术实现架构

### 5.1 支付系统

```
用户 → Stripe/微信支付 → 订单确认 → 后端更新 tier → 前端解锁功能
                                    ↓
                              数据库 user_tier 表
                              { user_id, tier, expires_at, billing_cycle }
```

**选型考虑**：
- 国内用户：微信支付 / 支付宝（必须支持）
- 海外 API 客户：Stripe（信用卡）
- 混合方案：前端检测地区 → 路由支付渠道

### 5.2 功能权限控制

```
app.js → checkTier(userId) → 返回 { free/pro/team/api }
                              ↓
                    根据 tier 限制：
                    - 题库浏览次数
                    - 分析报告生成次数
                    - AI 审核访问
                    - 上传次数
                    - 导出按钮可见性
```

**实现方式**：
- 后端 middleware 校验 tier + 用量计数
- 前端根据 tier 渲染 UI（隐藏/禁用超限功能）
- Redis 计数器追踪每日用量（题库浏览/分析/上传）

### 5.3 API 计费系统

```
API 客户 → API Gateway (Rate Limit + Auth) → 后端处理 → 返回数据
                                              ↓
                                        用量计数器 (Redis)
                                        ↓
                                    月末汇总 → 生成账单 → 支付渠道
```

**API Gateway 选型**：
- 自建：Flask + Redis rate limiter（初期低成本）
- 成熟期：Kong / AWS API Gateway（高可用）

## 8. ADE 协同的关键设计原则

1. **免费层不作降级**：核心浏览功能永久免费，不设期限。降级会产生负面口碑，阻碍获客。

2. **Pro 核心卖点是 AI 审核**：这是免费层无法体验的强差异化功能。教师最痛的需求是"审核题目质量"，AI 自动审核是刚需。

3. **数据退出机制是合规关键**：免费用户可开启 Privacy Mode 退出数据贡献。这是个人信息保护法的合规要求，也是建立信任的基础。

4. **API 是利润引擎而非副线**：到 M30，API + 数据包贡献 77% 利润。这意味着产品设计需要从第一天就考虑数据结构化。

5. **月付优先于年付**：初期用月付降低决策门槛，等用户粘性形成后推年付折扣。不建议首推年付。

6. **阶梯定价而非功能阉割**：Team 版不是 Pro + 几个功能，而是 Pro + 协作层。API 不是 Pro + 接口，而是全新产品线。每层有独立价值主张。
